# ReadMe
### 何？
* jpxが発表する投資部門別売買動向をjpxのAPIで取得する。
* 昔は「投資主体別売買動向」など。
* update_bumon.jsが木金月火の18時台に稼働して情報を取得する。
* dataフォルダ内のcsvを都度、上書きする。
* csvをindex.htmlで閲覧する。

### 情報発表タイミング
* 投資部門別情報・週次の第4営業日＝木曜日の18:00頃
* 通常は木曜日、祝日等非営業日がある場合はその分後ろ倒し。
* 連休等により通常と異なる公表スケジュールとなる場合はこちら。
* https://www.jpx.co.jp/markets/statistics-equities/investor-type/index.html

### 政府統計（日銀API・最下段チャート）
* `update_boj.js` が日本銀行「時系列統計データ検索サイト」APIから政府統計（為替・金利・マネー・短観・物価等）を取得する。
* 階層API(getDataLayer)でDBを丸ごと取得し、`data/boj/<DB>_<期種>.csv`（ワイド形式）と `data/boj/_catalog.csv` を出力する。
* `bojlist.csv` でチャートのプルダウンに出す系列を厳選できる（空なら _catalog.csv の全系列を自動表示）。列に集約方法(last=期末値 / sum=合計)を指定可能。
* APIキー不要。クレジット表示はindex.html下部に記載。高頻度アクセス禁止のためリクエスト間隔を空けている。
* `.github/workflows/update_boj.yml` で取得（当面は手動実行のみ。安定後に毎週金曜20:00の自動化を有効化予定）。

### 時間軸の粒度切替
* 画面上部の「日次／週次／月次」ボタンで全4段チャートの時間軸を連動切替する。
* 集約ルール：水準系（株価・為替・金利等）＝期末値、フロー系（部門別差引等）＝期間合計。

### 政府統計（e-Stat・政府統計①②チャート）
* `update_goj.js` が e-Stat 本体API(getStatsData ver3.0)から、`gojlist.csv` に列挙した系列だけを取得し、`data/goj/<ファイル>.csv`（ワイド形式）と `data/goj/_catalog.csv` を出力する。`gojlist.csv` が「収集」と「表示」の単一ソース。
* `update_kinro.js` が毎月勤労統計（e-Stat 本体APIのDBが2016年で凍結のため、長期時系列表「実数・指数累積データ」のファイル配信CSV）を取得し `data/goj/kinro.csv` を生成、`_catalog.csv` に追記する。
* 貿易・外需（国際収支系）は e-Stat 本体APIに無いため、日銀API側（`bojlist.csv` → `data/boj/BP01_M.csv`）で取得している。
* appId が必要（GitHub Actions の Secret `ESTAT_APPID`）。高頻度アクセス回避のためリクエスト間隔を空け、一過性のネットワーク障害には指数バックオフで再試行する。

#### 収集タイミング（GitHub Actions・週次）
* GOJ（`update_goj.js`＋`update_kinro.js`）: 毎週土曜 **JST 06:00**（cron `0 21 * * 5`）。
* BOJ（`update_boj.js`）: 毎週土曜 **JST 08:00**（cron `0 23 * * 5`）。
* 対象統計はすべて月次・四半期で、公表は月内に分散するため週1回で最長7日以内に最新月を反映できる。手動実行(workflow_dispatch)も可。
* 主な公表ラグの目安: CPI ≈3週 / 鉱工業指数 ≈4週 / 労働力調査・家計調査 ≈1か月 / 毎月勤労統計 ≈5週 / 景気動向指数・国際収支 ≈5〜6週 / 建築着工 ≈1か月 / 法人企業統計 ≈2.5か月 / GDP(QE) ≈6週。

#### 未収集（参考）
* ソフトウェア投資額: 法人企業統計に単独系列が無く「設備投資合計−ソフトウェア除く設備投資」の差分が必要なため未収録。
* 企業倒産件数・倒産負債総額: 民間調査(東京商工リサーチ等)で e-Stat に政府統計として存在しないため未収録。

#### 政府統計チャートのデフォルト表示
* 政府統計①「物価と賃金」: 総合CPI / コアCPI / 現金給与総額 / 実質賃金
* 政府統計②「景気・生産・雇用」: 実質GDP / 鉱工業生産指数 / CI一致指数 / 完全失業率
* 複数選択時は起点100で正規化して比較する。

#### 注意（時間軸の正規化）
* 月次・四半期・年度・暦年が混在する統計表では、系列ごとに最も細かい頻度のみを採用する（年次値が12月の月次値を上書きするのを防止）。
* 四半期ラベルは「2026年1 - 3 月」「2025年1～3月期」等の形式があり、開始月から四半期を判定して期末月(03/06/09/12)に正規化する。
